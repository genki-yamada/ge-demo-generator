# GE Demo Generator 使い方ガイド

## このサービスは何？

**GE Demo Generator** は、お客様の業務課題を入力するだけで、
**Gemini Enterprise** 向けの「動くデモ環境」を自動生成する Web ツールです。

入力された業務シナリオに合わせて、以下を自動で用意します。

- **データセット**（BigQuery のテーブル、Firestore のドキュメント）
- **AI エージェント**（MCP ツール群を備えた Gemini エージェント）
- **リアルタイム運用ダッシュボード**（Data Viewer）

数週間かかっていたデモ準備を、数分で完了できるのが特徴です。

> 本体は Google Apps Script（GAS）上で動く Web アプリです。

---

## 全体の流れ（ざっくり）

利用には大きく2つの立場があります。

| 立場 | やること | 章 |
|---|---|---|
| **管理者** | ツール本体を一度だけセットアップして公開する | A |
| **利用者** | 公開された Web UI からデモを生成して動かす | B |

すでに管理者がセットアップ済みなら、利用者は **B章だけ** 読めば使えます。

---

# A. 管理者：ツール本体のセットアップ（初回のみ）

> ツールを公開する人だけが行う作業です。一度だけ実施します。

## A-1. 事前準備

- [Node.js](https://nodejs.org/) をインストール
- Clasp をインストール：`npm install -g @google/clasp`
- **課金が有効な Google Cloud プロジェクト**
- 以下の Google Cloud API を有効化
  - Vertex AI API
  - BigQuery API
  - Google Drive API
  - Sheets API
  - Maps Platform（Places / Geocoding / Routes）

## A-2. コードを取得して準備

```bash
git clone https://github.com/ryotat7/ge-demo-generator.git
cd ge-demo-generator
npm install
clasp login
```

## A-3. Apps Script プロジェクトを作る

1. [script.google.com](https://script.google.com) で新規プロジェクトを作成
2. **プロジェクト設定（歯車）→ ID** から **スクリプト ID** をコピー
3. プロジェクト直下に `.clasp.json` を作成

   ```json
   {"scriptId": "あなたのスクリプトID"}
   ```

## A-4. コードを反映（デプロイ）

```bash
clasp push     # ローカルのコードを Apps Script に送る
clasp open     # ブラウザでエディタを開く
```

## A-5. Google Cloud プロジェクトと紐付け

Apps Script エディタ → **プロジェクト設定（歯車）**
→ **Google Cloud Platform プロジェクト** → **プロジェクトを変更**
→ **プロジェクト番号**（ID ではない）を入力。

## A-6. スクリプトプロパティを設定（必須）

このツールは設定値をコードに直書きしません。すべて **スクリプトプロパティ** で管理します。

| プロパティ | 説明 |
|---|---|
| `PROJECT_ID` | Google Cloud プロジェクト ID（例：`my-project-123`） |
| `LOG_SHEET_URL` | 利用ログ用スプレッドシートの URL（`Usage_Logs` シートを含むこと） |

任意設定（省略可）：

| プロパティ | 既定値 | 説明 |
|---|---|---|
| `LOCATION` | `global` | Vertex AI のロケーション |
| `MODEL` | `gemini-3.5-flash` | データ生成に使う Gemini モデル |

設定方法（おすすめ）：Apps Script エディタで `initializeProject` 関数を実行。

```javascript
initializeProject('あなたのプロジェクトID', 'https://docs.google.com/spreadsheets/d/xxx/edit');
```

> `PROJECT_ID` か `LOG_SHEET_URL` が未設定だと、本体画面ではなく設定エラー画面が表示されます。

## A-7. 権限の承認（初回のみ）

エディタで `forceAuthorizeSpreadsheet` 関数を選び **実行（▶）**。
表示される「権限の確認」に従って承認します。
（「このアプリは確認されていません」と出たら **詳細 → 移動** で進めます）

## A-8. 利用ログ用スプレッドシートを用意

1. 新規スプレッドシートを作成
2. `Usage_Logs` という名前のシートを作る（ヘッダー行は自動で整えられます）
3. その URL を `LOG_SHEET_URL` プロパティに設定

## A-9. Web アプリとして公開

Apps Script エディタ → **デプロイ → 新しいデプロイ** → 種類は **ウェブアプリ**。

- **次のユーザーとして実行**：`ウェブアプリにアクセスしているユーザー`
- **アクセスできるユーザー**：`全員`（必要に応じて制限）

→ **デプロイ** して表示される **Web アプリ URL** が、利用者に配る URL です。

# B. 利用者：デモを生成して動かす

> 管理者が公開した Web アプリ URL を開くところから始めます。
> 操作の多くは **Cloud Shell**（Google Cloud のブラウザ上のターミナル）で行います。

## B-1. デモを生成する

1. 配布された **Web アプリ URL** を開く
2. お客様の **業務課題やシナリオ** を入力
3. ツールが自動でデータとエージェント構成を設計し、
   専用の **セットアップスクリプト**（`setup-demo-xxx.sh`）を生成

## B-2. 環境を構築する（Cloud Shell）

1. Web UI の **Step 3: Deploy** で **Setup Script** の **Copy** を押す
2. Cloud Shell のターミナルに貼り付けて **Enter**

> このスクリプトが、あなたの GCP プロジェクトに実際の環境を作ります
> （BigQuery データセット、Firestore、Data Viewer、エージェント一式）。

## B-3. エージェントを起動する

スクリプト完了後、画面に起動コマンドが表示されます。手動で起動する場合：

```bash
# 生成されたデモフォルダの adk_agent に移動
cd $(ls -d demo-*/ | head -n 1)adk_agent

# エージェントを起動
../.venv/bin/adk web --allow_origins="*"
```

## B-4. 画面を開いて試す

1. `Uvicorn running on http://127.0.0.1:8000` と出たら、
   Cloud Shell 右上の **ウェブでプレビュー → ポート 8000 でプレビュー**
2. 新しいタブで **app** を選び、新規セッションを開始
3. プロンプト例：
   - 「BigQuery ツールで売上トレンドを分析して」
   - 「item #104 のメンテ状況を更新して Firestore にログを記録して」

### （任意）リアルタイム Data Viewer

```bash
cd viewer_app
python3 -m http.server 8080
```

→ **ウェブでプレビュー → ポート 8080** でダッシュボードを表示。

## B-5.（任意）Gemini Enterprise に登録

```bash
# adk_agent ディレクトリで実行
uvx agent-starter-pack enhance     # 質問はすべて既定値でOK
make deploy                        # Cloud Run にデプロイ
make register-gemini-enterprise    # Gemini Enterprise に登録
```

## C. 困ったとき（よくあるトラブル）

| 症状 | 対処 |
|---|---|
| 最初の応答が遅い | Cloud Run のコールドスタート。もう一度同じプロンプトを送ると安定します |
| 403 / 権限エラー | Cloud Run のサービスアカウントに必要なロールを付与（`tutorial.md` の手順4参照） |
| Data Viewer のデプロイ失敗 | 組織ポリシーの制限が原因のことが多い。ローカルの `adk web` でプレビュー可能 |
| 認証スコープ不足エラー | `gcloud auth application-default login` を再実行 → エージェント再起動 |

---

## D. 後片付け（クリーンアップ）

作ったデモ環境はスクリプトに `--cleanup` を付けて一括削除できます。

```bash
bash setup-demo-xxx.sh --cleanup
```

BigQuery データセット、Maps API キー、Cloud Run サービス、Firestore、
Gemini Enterprise 登録、各種シークレットなどがまとめて削除されます。
（削除前に確認プロンプトが出ます）
